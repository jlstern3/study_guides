Git Commands Test

Type your answers to each function in the respective input box. When necessary, refer to a file name as: file_name and a directory/folder name as: folder_name.

  1. Initialize a repository (git init)
  2. Stage all files that were changed since last back up (git add .)
  3. Show all files that were changed since last backup and which ones were already added to staging area (git status)
  4. Commit changes to repo with message (git commit -m "message here"
  5. Show all git branches (git branch)
  6. Switch to branch name provided in git repo/create new branch (git branch branch_name_here)
  7. Show all backups created in repo (git log)
  8. Add a remote place called 'origin' to a remote URL (git remote add origin URL_name_here)
  9. Push changes in your local repo to your remote repo (git push)
  10. Pull changes from remote repo to your local repo (git pull)
  11. Clone a remote repo to your local folder (git clone ____)

Defining Terms

  1. Fork (a remote, server-side copy of a repo -- distinct from the original)
  2. Clone (local copy of a remote repo)
  3. *** So forking a repo that you don't own creates a copy of that repo on your GitHub profile. You can then clone that fork to get a copy of the repo on your local machine.
